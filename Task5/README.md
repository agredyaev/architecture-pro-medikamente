# Task5 — план переключения миграции

## 1. Цель

Выбрать стратегию миграции для «Медикаменте» и зафиксировать cutover-план с ролями, рисками, мерами снижения и откатом.

## 2. Выбранная стратегия

Выбран `параллельный запуск`.

Причины:

- Текущий ландшафт состоит из Excel, файлов, `1С` и ручных операций, а не из одного унаследованного приложения.
- Главный риск связан с данными: patient ID, consent, аудит, платежи, лабораторные результаты.
- Для медданных и платежей нужен проверяемый откат и сверка старого и нового контуров.
