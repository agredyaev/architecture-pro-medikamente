# Автоматизированные контроли

## 1. Реестр контролей

| ID | Контроль | Триггер | Правило | Реакция системы | Доказательство |
| --- | --- | --- | --- | --- | --- |
| `AC-01` | Обязательное тегирование и классификация | Создание или изменение записи, документа, результата, выгрузки | Сохранение разрешено только при наличии `tag.class`, `tag.domain`, `tag.subject_scope`, `tag.legal_basis`, `tag.retention`, `tag.masking`, `tag.export`, `tag.audit` | Операция блокируется, событие пишется в аудит | Запись в audit trail, статус проверки |
| `AC-02` | Проверка legal basis, consent и прав доступа | Чтение, изменение, экспорт, внешняя передача | Операция разрешена только при `role=allow`, `patient_scope=allow`, `legal_basis=allow` | Операция запрещается, пользователю возвращается отказ без раскрытия данных | Policy log, audit event |
| `AC-03` | Контроль схемы и минимизации полей | Внешний или межсервисный API-вызов | Контракт не должен содержать лишние поля и запрещенные классы данных | Payload отклоняется, генерируется событие нарушения контракта | API gateway log, contract validation result |
| `AC-04` | Контроль состояния шифрования и ключей | Плановая проверка и создание ресурса | БД, объекты, backup и очереди обязаны использовать шифрование; ключи должны ротироваться по регламенту | Ресурс помечается как non-compliant, создается алерт | Compliance report, KMS status |
| `AC-05` | Controlled export и DLP | Экспорт файла, отчета или выгрузки | Запрещены объекты с `tag.export=forbidden`; агрегаты экспортируются только через разрешенный сервис | Экспорт блокируется или маршрутизируется через controlled export | Export log, DLP incident |
| `AC-06` | Полнота и неизменяемость аудита | Любое чтение, изменение, удаление, экспорт, интеграционный вызов | Событие должно попасть в централизованный audit store без потери полей контроля | При отсутствии события формируется инцидент и технический алерт | Audit completeness report |
| `AC-07` | Выявление аномалий доступа | Поток audit-событий | Массовое чтение, массовый экспорт, нетипичное время доступа и нетипичный объем считаются подозрительными | Создается алерт ИБ и кейс расследования | SIEM correlation event |
| `AC-08` | Retention, delete и stop processing | Достижение срока хранения или запрос субъекта | По событию retention/request выполняется ровно одно действие: `archive`, `delete` или `set stop_processing` | Lifecycle job исполняет действие и пишет событие | Lifecycle log, delete journal |
| `AC-09` | Аналитический gate перед загрузкой | Загрузка в raw or restricted zone, публикация витрины | Набор должен пройти классификацию, псевдонимизацию или обезличивание до публикации | Загрузка или публикация блокируется | Classifier result, ingestion log |
| `AC-10` | Контроль TLS, mTLS и сертификатов | Установление канала связи и плановая проверка | Внутренние и внешние каналы обязаны использовать разрешенные протоколы и валидные сертификаты | Соединение отклоняется или создается предупреждение до истечения срока | TLS scan, certificate status |
| `AC-11` | Контроль log redaction и trace sanitization | Запись лога, трейса, dump или support bundle | Чувствительные поля не должны попадать в техартефакты в открытом виде | Запись маскируется, нарушение уходит в SIEM | Sanitization report, DLP incident |
| `AC-12` | Контроль endpoint-копий и неразрешенных каналов | Скачивание файла, запись на USB, отправка почтой, синхронизация в локальную папку, печать | Передача чувствительных объектов разрешена только через approved export path | Операция блокируется или переводится на controlled export с инцидентом | DLP log, EDR event |
| `AC-13` | Контроль ротации секретов и восстановления резервных копий | Плановая проверка ключей и drill восстановления | Секреты и сертификаты не должны быть просрочены; backup должен проходить restore test и immutability check | Создается инцидент несоответствия и блокируется выпуск критичных изменений | Rotation report, restore drill result |
| `AC-14` | Контроль masking enforcement | Формирование UI-ответа, API-ответа или пользовательского экспорта | Поля с `tag.masking=partial/full` должны выдаваться только в маскированном виде вне разрешенной privileged role | Ответ блокируется или маскируется автоматически | UI snapshot test, API contract test, export validation report |
| `AC-15` | Контроль полноты lineage | Загрузка, трансформация, публикация витрины, лабораторный обмен, controlled export | Публикация или выдача разрешены только при наличии `lineage` с `source`, `transform`, `timestamp`, `recipient` | Публикация или выдача блокируется до записи lineage | Lineage completeness report |

## 2. Минимальные показатели контроля

| Метрика | Целевое значение | Назначение |
| --- | --- | --- |
| Доля объектов с обязательными тегами | `100%` | Исключить неуправляемые данные |
| Доля чувствительных API-вызовов с contract validation | `100%` | Исключить лишние поля и несанкционированную передачу |
| Доля чувствительных хранилищ с включенным шифрованием | `100%` | Исключить хранение в открытом виде |
| Полнота audit-событий для чтения, изменения и экспорта | `100%` | Обеспечить доказуемость доступа |
| Доля аналитических публикаций через классификационный gate | `100%` | Исключить публикацию неразмеченных наборов |
| Доля техартефактов с примененным redaction | `100%` | Исключить утечки через логи, трейсы и dump |
| Доля чувствительных экспортов, прошедших через controlled export | `100%` | Исключить ручные вложения и локальные копии вне контроля |
| Доля резервных копий, прошедших restore drill по графику | `100%` | Подтвердить восстановимость и корректность ключей |
| Доля ответов с обязательным masking enforcement | `100%` в окне `30d` | Исключить выдачу чувствительных полей в открытом виде |
| Доля наборов и результатов с полным lineage | `100%` в окне `30d` | Обеспечить прослеживаемость источника, преобразования и экспорта |

## 2.1 Пороги и окна обнаружения аномалий

| Сценарий | Порог | Окно измерения | Источник |
| --- | --- | --- | --- |
| Массовое чтение медданных одним пользователем | `>50` карточек | `15m` | Audit trail |
| Массовый экспорт отчетов | `>3` экспорта | `1h` | Export log |
| Нетипичное время доступа | Любое чтение `22:00-06:00` для не-duties ролей | `realtime` | SIEM correlation |
| Резкий рост ошибок contract validation | `>20` ошибок | `10m` | API gateway log |

| Проблемная зона | Контроли |
| --- | --- |

| Отсутствие masking и неполный lineage | `AC-14`, `AC-15` |
